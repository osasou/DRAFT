とりあえず、 CHIRRUP_good_1_slot_rev* はrev8で終了。完成。ということにする。

次は、この CHIRRUP_good_1_slot_rev8 とEMアルゴリズムを合わせていく。
プロジェクト名は DRAFTディレクトリ配下の DRAFT_0_0 にする。

と CHIRRUP_good_1_slot_rev8 の readme.txt で書いたが、

DRAFT_0_0 では、
まず
・h_distance.m を削除
・run.m と tester.m の返り値をいじって、h_hat の全ての値を tester.m で h_all_output として扱うことができるようにした。
        figure
        plot(h_all_output,'o')
        で、h_all_outputをプロットして、グラフが出るようにした。
        なお、これまでのグラフはコメントで出ないようにしておいた。

通信路の推定値 h_dash が時々、とても大きいことがあるため、check_h_dist.m (h_hat のそれぞれの値の大きさが5以上であるかをチェックする関数) を作成し、Decoder.m でチェックした。

各ユーザの通信路 h の値はそれぞれ別の値で固定するという仮定であるから、
Encoder.m において、
                if (k == 1)
                    h = 1;
                elseif (k == 2)
                    h = 1i;
                elseif (k == 3)
                    h = 0.5 + 0.5i;
                elseif (k == 4)
                    h = -0.1 + 0.3i;
                else
                    h = -0.8 - 0.4i;
                end 
                として、ユーザごとに通信路 h の値を固定した。


これまで、h_hat の実部と虚部の比が関係あると言ってきたが、実際にグラフに表すと、関係があると言えることがわかる。実際に、傾きが一定の一次関数になっていることがわかる。

--> 傾き(実部と虚部の比)において EMアルゴリズムを適用させれば、クラス分けができる。

ちなみに、自分で作成した EMアルゴリズムのプログラムはクラスが2つの時しか動かないから、それを解決するか、ライブラリを使う。
